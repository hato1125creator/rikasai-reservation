# 梨花祭 入場管理システム（nyuujou-system）完全マニュアル

このマニュアルは、このシステムを将来の世代（プログラミング初心者）がゼロから復元したり、AI（ChatGPTなど）に依頼して再構築したり、エラーが起きた際に対処できるようにするためのドキュメントです。

---

## 📂 1. システムの全体構成

このシステムは、文化祭（梨花祭など）の入場予約・受付を管理するためのウェブアプリケーションです。特別なデータベースソフト（MySQLなど）を使わず、JSONファイルにデータを保存する構成になっており、管理・移行が簡単なのが特徴です。

*   **バックエンド（サーバー側）**: Node.js + Express
*   **データベース**: ローカルJSONファイル（`server/database.json`）
*   **フロントエンド（画面側）**: HTML, CSS, JavaScript (Vanilla JS)
*   **主な機能**:
    *   **ゲスト予約**: 招待コードを使って予約し、QRコード付きの受付メールを自動送信。
    *   **生徒用パネル**: メールアドレスでOTP（ワンタイムパスワード）認証し、自分の招待枠（ゲストスロット）を管理・リンク発行。
    *   **管理者ダッシュボード**: 予約者のステータス（入場済みか等）の変更、CSVからの招待コード一括生成、時間帯別入場レポートの表示。
    *   **受付QRリーダー**: 発行されたQRコードを読み取り、入場処理（チェックイン）を自動化。

## 🤖 2. AIに作ってもらう（再構築する）ためのプロンプト（指示書）

もしシステムが完全に消えてしまい、AI（ChatGPTやClaudeなど）にゼロから作り直してもらいたい場合は、以下の文章（プロンプト）をそのままAIにコピー＆ペーストして伝えてください。

> 【AIへの指示書】
> 文化祭の入場管理システムをNode.jsとExpressで構築してください。要件は以下の通りです。
> 
> **[技術スタック]**
> - サーバー: Node.js, Express
> - データベース: SequelizeのようなORMのインターフェースを持つが、裏側はただのJSONファイル（`database.json`）に保存するカスタムモジュールを作成（SQLiteやMySQLは使わず、ファイル読み書きで完結させること）。
> - フロントエンド: プレーンなHTML, CSS, JavaScript（React等は不要）。
> 
> **[ディレクトリ構成案]**
> - `app.js` (サーバーのメインファイル)
> - `server/db-local.js` (JSONファイルへ保存DB処理のモジュール)
> - `server/public/` (公開用フォルダ: `index.html`, CSS, JS)
> - `server/public/guest/` (ゲスト用予約画面や確認画面)
> - `server/public/student/` (生徒用画面)
> 
> **[必要な機能]**
> 1. **予約機能**: ユーザーは招待コードを入力し、名前や連絡先を登録。成功するとQRコード付きの確認メール（Nodemailerを使用）が送信される。
> 2. **招待コード管理機能**: 管理者はシステムから招待コードを生成（CSV読み込み対応）できる。
> 3. **QRコード受付**: 当日、ユーザーが持ってきたQRコードを管理者が読み取ることで「チェックイン済み」ステータスに変更される。
> 4. **生徒枠（ゲストスロット）管理**: 生徒は学校のメアドでOTP認証し、自分に割り当てられた招待枠（URL）を発行できる。
> 5. **セキュリティ**: レート制限（express-rate-limit）、CORS設定、JWTを使った管理者認証を実装する。
> 
> この要件に基づき、まずは `package.json` と `server/db-local.js` 、`app.js` のベースコードを書いてください。

## 🛠️ 3. ゼロから立ち上げる手順（初心者向け）

もしソースコード（ファイル一式）が手元にある場合は、以下の手順でシステムを起動できます。

### Step 1. Node.js のインストール
パソコンに [Node.js](https://nodejs.org/ja/) をインストールします（LTS推奨）。インストールできたら、コマンドプロンプト（またはターミナル）を開き、`node -v` と入力してバージョンが出れば成功です。

### Step 2. 依存関係のインストール
ソースコードがあるフォルダ（`package.json` がある場所）でコマンドプロンプトを開き、以下のコマンドを実行します。
```bash
npm install
```
※これにより、`node_modules` フォルダが自動作成され、必要な部品が入ります。

### Step 3. 環境変数（.env）の設定
フォルダの中に `.env` というファイルを作成し、以下の情報を記述します。メール送信やセキュリティのためのパスワードのようなものです。

```env
PORT=3007
JWT_SECRET=your_super_secret_key_here
EMAIL_USER_1=your_email@gmail.com
EMAIL_PASSWORD_1=your_app_password
EMAIL_USER_2=your_backup_email@gmail.com
EMAIL_PASSWORD_2=your_backup_app_password
FRONTEND_URL=http://localhost:3007
```
*(※Gmailを使う場合は、Googleアカウント設定から「アプリパスワード」を取得して設定してください)*

### Step 4. システムの起動
以下のコマンドでサーバーを起動します。
```bash
node app.js
```
または
```bash
npm start
```
「Database synced (JSON)」と表示されれば成功です。ブラウザを開き `http://localhost:3007` にアクセスしてください。

## 🆘 4. よくあるエラーと解決方法

後輩たちがシステムを運用する際によく起きるエラーの対処法です。

### 🚨 エラー1: `Error: Cannot find module 'express'`
**【原因】** モジュール（部品）がインストールされていません。
**【解決策】** コマンドプロンプトで当該フォルダを開き、`npm install` を実行してください。

### 🚨 エラー2: メールが送信されない（QRコードが届かない）
**【原因】** `.env` ファイルのメール設定が間違っているか、Googleアカウントの「アプリパスワード」が無効になっています。
**【解決策】**
1. `.env` ファイルの `EMAIL_USER_1` と `EMAIL_PASSWORD_1` を確認する。
2. 通常のGoogleパスワードではなく、「2段階認証」をオンにした上で作成する「アプリパスワード（16桁）」を使用しているか確認する。

### 🚨 エラー3: 予約しようとすると「無効な招待コード」と出る
**【原因】** 入力した招待コードが `database.json` に存在しないか、すでに「使用済み（used: true）」になっています。
**【解決策】** 管理者用画面から新しい招待コードを発行するか、データベース（`server/database.json`）をテキストエディタで直接開き、該当コードの `"used": true` を `false` に変更して保存し直してください。

### 🚨 エラー4: サーバーが起動しない / `EADDRINUSE` というエラーが出る
**【原因】** ポート番号（3007番）がすでに別のプログラムで使われています。（あるいは、システムを二重に起動しようとしています）
**【解決策】**
1. すでに開いている黒い画面（コマンドプロンプト）があれば `Ctrl + C` を押してシステムを一度終了させます。
2. それでもダメな場合、`.env` ファイルの `PORT=3008` のように別の番号に変更して `node app.js` を実行します。

### 🚨 エラー5: パスワードを忘れて管理者画面に入れない
**【解決策】** 
初期設定では、IDは `admin`、パスワードは `rikasai123` です。
変更してしまってログインできない場合は、ファイルを直接いじってリセットできます。
1. `server/database.json` を開きます。
2. `"users"` の部分を探し、中身ごと削除して空配列 `[]` にします。
3. サーバーを再起動（`Ctrl + C` してから `node app.js`）すると、初期管理者（ID: admin, PW: rikasai123）が自動で再作成されます。

## 📸 5. システムの使い方（操作マニュアル）

ここからは、実際にシステムを動かして操作する手順を説明します。
*(※以下の `![説明](パス)` の部分に、後から実際のスクリーンショット画像を保存してパスを書き換えると画像が表示されます。)*

### 5-1. 管理者画面へのログインと初期設定
1. ブラウザで `http://localhost:3007/admin`（または本番用URL）にアクセスします。
2. ID（初期は `admin`）とパスワード（初期は `rikasai123`）を入力してログインします。

![管理者ログイン画面のスクショをここに貼る](./screenshots/admin_login.png)

3. ログイン後、**ダッシュボード**が表示されます。ここでは現在の予約数や入場者数のグラフなどが確認できます。

![ダッシュボード画面のスクショをここに貼る](./screenshots/admin_dashboard.png)

### 5-2. 招待コードの発行（CSV一括・個別）
ゲストが予約するためには「招待コード」が必要です。
1. 管理者メニューから「招待コード管理」などを開き、**CSVアップロード** または **個別生成** を選びます。
2. 生徒のリスト（CSVファイル）をアップロードすると、自動的に「保護者用1枚・ゲスト用3枚」などのコードが生成され、新しいCSVとしてダウンロードされます。

![招待コード生成画面のスクショをここに貼る](./screenshots/generate_codes.png)

### 5-3. ゲストによる入場予約
1. ゲスト（一般参加者や保護者）には、生徒から招待コードを伝えてもらいます。
2. ゲストは `http://localhost:3007/guest`（ゲスト用予約ページ）にアクセスします。
3. 自分の名前、メールアドレス、そして**招待コード**を入力して「予約する」ボタンを押します。

![ゲスト予約画面のスクショをここに貼る](./screenshots/guest_reservation.png)

4. 予約が成功すると、入力したメールアドレス宛に**入場用QRコード付きのメール**が届きます。

### 5-4. 生徒用パネル（OTPログインとリンク発行）
生徒自身が、自分の招待枠（ゲストスロット）を管理する画面です。
1. 生徒は `http://localhost:3007/student` にアクセスし、学校から配布されたメールアドレスを入力します。
2. メールに届いた「ワンタイムパスワード（OTP）」を入力してログインします。

![生徒ログイン・OTP画面のスクショをここに貼る](./screenshots/student_login.png)

3. ログイン後、自分の招待できるゲスト枠のURLを発行し、LINE等で友だちに送ることができます。友だちはそのURLから直接予約・QRコード取得が可能です。

### 5-5. 当日の受付（QRコードリーダーによるチェックイン）
文化祭当日は、この機能を使って受付をスムーズに行います。
1. 受付担当のスマホやタブレットで `http://localhost:3007/admin/qr-reader`（または専用の読み取り画面）を開きます。
2. ゲストが提示したメールのQRコードをカメラで読み取ります。
3. 画面に「入場承認」と表示されればOKです！自動的にステータスが「入場済み」に切り替わります。

![QR読み取り・承認画面のスクショをここに貼る](./screenshots/qr_reader.png)

## 📘 6. データのバックアップについて

このシステムはデータベースとして `server/database.json` というただのテキストファイルを使っています。
**文化祭当日や前日には、必ずこの `database.json` ファイルをコピーしてバックアップ（USBメモリやGoogle Driveなど）を取っておいてください。** もしデータが消えても、バックアップしたファイルを上書きするだけで完全に復元できます。
